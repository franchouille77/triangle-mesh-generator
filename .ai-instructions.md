## 1. Overview

**Triangle Mesh Generator** is a single-file HTML web application (`index.html`) that generates an equilateral triangle mesh by drawing 3 families of parallel lines (0°, +60°, −60°) inside a rectangle W × H.

- No external dependencies, no bundler, no framework.
- Works via `file://` — no ES6 modules, no fetch.
- All CSS, HTML, and JS are inline in a single file (~1800 lines).
- SVG rendering with viewBox in millimeters.
- Full English interface and SEO content.
- Designed for GitHub Pages deployment at `https://trianglemeshgenerator.github.io/`.

## 3. Core Concept — Line Thickness and Interior Side

**This is the most important concept for any modification.**

The user enters a side `c` that represents the **interior** dimension of the triangle: the distance between the **inner edges** of the lines forming each side. It is NOT the axis-to-axis distance.

### Essential Formulas

```
d  = c × √3/2 + 3e/2    Inter-axis spacing (perpendicular distance)
h  = c × √3/2            Interior triangle height
sh = 2e / √3              Horizontal cross-section at diagonal intersections
```

**Geometric explanation:**
- 3 families of parallel lines at 60° apart create equilateral triangle tessellation.
- If perpendicular spacing between consecutive axes is `d`, the axis-to-axis triangle side is `2d/√3`.
- When lines have thickness `e`, each inner edge recedes by `e/2` perpendicular to its line.
- Interior triangle height is NOT `d − e` but `d − 3e/2`, because:
  - The horizontal side rises by `e/2` (direct reduction)
  - The opposite vertex (intersection of 2 inner diagonal edges at 60°) drops by `e`
  - Total height reduction = `3e/2`
- For interior side = `c`: `d − 3e/2 = c × √3/2` → `d = c × √3/2 + 3e/2`.

### SVG Rendering Consequence
The `stroke-width` of `<line>` elements is set to `e` (actual thickness in mm), since the viewBox is in mm. When `e = 0`, a thin proportional stroke is used for visualization (`max(W,H) × 0.003`).

## 4. Automatic Origin Offset

`computeOriginOffset(e)` calculates an offset applied to all 3 line families so the **inner vertex of the first triangle** (top-left corner) aligns exactly with the origin (0, 0).

| Offset | Formula | Value (e=4) |
|--------|---------|-------------|
| dx | `−e√3/2` | −3.464 mm |
| dy | `−e/2` | −2.0 mm |

### Data flow

```
User inputs (W, H, c, e, dx, dy)
         │
         ▼
    draw()
    ├── computeSpacing(c, e) → d
    ├── getOriginOffset() → {dx, dy}
    ├── computeTrimRight/Bottom(d, shift, e, L/H) → trim data
    ├── For each family (0°, +60°, −60°):
    │   └── generateFamily(angle, d, clipRight, clipBottom, dx, dy, e)
    │       ├── Calculate visible line indices (projection onto normal)
    │       ├── Filter tangencies (lines that merely touch edges)
    │       └── For each line: clipLine() + butt cap extension
    ├── Inject SVG (<rect>, <line>, trim lines, origin marker)
    ├── buildDimAnnotations(L, H)
    ├── buildMeshDimAnnotations(families, d, c, e, ...)
    └── Update info-bar (d, h_int, sect.h, line counts, trim info)
```

## 6. Features

### Trim Right (vertical cut line)
- Computes a vertical trim line using diagonal-axis crossing grid (`halfStep = sideAxis / 2`)
- Ensures mirror symmetry: distance from rect right edge to trim = distance from left edge to first crossing
- Mesh lines are clipped to `xLeftEdge` (not `L`)
- Color: magenta `#ff44ff` (or black when "Black lines" is active)
- `computeTrimRight(d, shift, e, L)` → `{xCenter, xLeftEdge, nSteps, nFull, isHalf, halfStep, sideAxis}`

### Trim Bottom (horizontal cut line)
- Finds the last horizontal-line axis whose stroke fits inside the rectangle (`y + e/2 ≤ H`)
- Mesh lines are clipped to `yTopEdge` (not `H`)
- `computeTrimBottom(d, shift, e, H)` → `{yCenter, yTopEdge, nSteps, step}`

### Gauge Buttons (📐)
- Two independent toggles: rectangle gauges and mesh gauges
- Rectangle: shows W and H dimensions with extension lines, ticks, and labels
- Mesh: shows edge length `c` on inner triangle side + stroke width `e` on first diagonal
- Also visible on hover/focus of corresponding toolbar section
- `rectGaugeEnabled`, `meshGaugeEnabled` flags

### Black Lines Mode (⬛)
- Forces all mesh lines + trim lines to `#000000` in:
  - Canvas rendering (`draw()`)
  - SVG export (`exportSVG()`)
  - DXF export (`exportDXF()` — color 7 for all layers)
- `blackLinesEnabled` flag

### Screen Calibration
- Dialog: 80vw × 80vh card (max 1000×750, min 320×300)
- Canvas draws a 5 mm white grid with cm ruler markings outside the grid area
- Ruler margin: top (horizontal cm labels) and left (vertical cm labels)
- Canvas background is transparent; only the grid area is white
- DPI slider (50–300), numeric input, Apply/Cancel buttons
- Keyboard: Enter = Apply, Escape = Cancel
- Important: overlay is shown BEFORE `drawCalibGrid()` runs (via `requestAnimationFrame`) so the container has real dimensions

### Scroll Controls
- Plain scroll = zoom (×1.12 per step, centered on cursor)
- Shift + scroll = horizontal pan
- Ctrl + scroll = vertical pan
- Click + drag = pan (uses `setPointerCapture` for reliable tracking)
- Double-click = fit all

### Export
- SVG: standalone file with viewBox in mm, lines grouped by family
- DXF: ASCII R12 format, layers per family + RECTANGLE + TRIM_RIGHT/TRIM_BOTTOM
- Y-axis inverted in DXF (Y-up convention: `dxfY = H − svgY`)
- Filenames: `triangle-mesh_YYYY-MM-DD_HHhMM.svg/.dxf`

## 7. SVG Coordinate System

- **viewBox** in mm: `(vb.x, vb.y, vb.w, vb.h)` — SVG units ARE millimeters.
- The working rectangle spans `(0, 0)` to `(W, H)`.
- Y-axis points down (standard SVG convention).
- Zoom: `vb.w` / `vb.h` control zoom level (smaller = more zoomed in).
- Pan: `vb.x` / `vb.y` control viewport offset.
- Zoom is centered on cursor position (not screen center).
- `fitAll()` shows the full rectangle with 6% margin, preserving aspect ratio.
- `setRealSize()` sets viewBox width to the physical screen width in mm.

## 9. Modification Rules

### Do NOT
- **Add external dependencies** (libraries, CDN) — the file must remain self-contained.
- **Use `import`/`export`** — incompatible with `file://`.
- **Modify the formula `d = c × √3/2 + 3e/2`** without understanding that `c` is the INTERIOR side and that `3e/2` comes from the non-perpendicular geometry at vertices.
- **Confuse axis-to-axis with edge-to-edge** — the entire logic relies on this distinction.

### Be careful
- Any change to spacing or origin offset affects all 3 families simultaneously.
- The viewBox is in mm — SVG coordinates ARE mm.
- `stroke-width` is in mm (= `e`), so lines scale with zoom.
- The rectangle outline uses `vector-effect: non-scaling-stroke` (always 1px on screen).
- Calibration grid: overlay must be visible before measuring container dimensions.
- Trim lines clip mesh to smaller boundaries — `clipRight` and `clipBottom` replace `L` and `H`.

### To add a feature
1. Identify the relevant section in the file (see §5).
2. Follow existing patterns (no framework, no ES6 classes, simple functions).
3. New UI controls go in `<header class="top-bar">` as a new `.ctrl-group` pill.
4. Toggle buttons use the `.offset-btn` class with `.active` state.
5. Geometric calculations must be documented with formulas.
6. Update the info-bar in `draw()` if the new quantity is relevant.
7. If the feature affects export, update both `exportSVG()` and `exportDXF()`.

## 11. Glossary

| Term | Meaning |
|------|---------|
| `c` | Interior triangle side (inner edge to inner edge) |
| `e` | Stroke width / line thickness |
| `d` | Perpendicular spacing between consecutive line axes |
| `h_int` | Interior triangle height = `c × √3/2` |
| `sect.h` | Horizontal cross-section at diagonal intersection = `2e / √3` |
| `shift` | Origin offset `{dx, dy}` — auto = `(−e√3/2, −e/2)` |
| `vb` | SVG ViewBox `{x, y, w, h}` in mm |
| `screenDPI` | Physical screen resolution (dots per inch) |
| `pxPerMm` | Pixels per mm = `DPI / 25.4` |
| `halfStep` | Half the axis-to-axis triangle side = `d / √3` (used by trim right) |
| `clipRight` | Right clipping boundary (L or trimRight.xLeftEdge) |
| `clipBottom` | Bottom clipping boundary (H or trimBottom.yTopEdge) |
| Cohen-Sutherland | Line clipping algorithm for a rectangle |
| Family | Set of parallel lines at a given angle (0°, +60°, −60°) |
| Butt cap | Flat stroke end — extended past rectangle edge so cap fully clears |
| JSON-LD | Structured data format for search engine rich results |
| OG | Open Graph protocol for social media sharing previews |
